# RSS 摘要 Skill

## Overview

自动拉取 153+ 个 RSS 源的最新文章，通过两轮 AI 筛选从海量文章中提取 Top 100 重要消息，按类别分组展示简短摘要。

## When to Use

**触发条件：**
- 用户想要快速了解今日重要新闻
- 需要从大量 RSS 文章中筛选重点
- 希望按类别查看各领域新闻

**使用方式：**
直接调用：`/rss`

## When NOT to Use

- 需要查看所有文章（使用 `rss-reader articles` 命令）
- 需要搜索特定关键词（使用 `rss-reader search` 命令）
- 需要手动筛选文章（使用 TUI 界面）

## Implementation

### 1. 检测平台与可执行文件

根据当前操作系统确定可执行文件路径：
- **Windows**: `.\target\release\rss-reader.exe`
- **Linux/macOS**: `./target/release/rss-reader`

检测方法：检查环境变量 `$OS` 是否包含 `Windows`，或使用 `uname` 判断。

项目根目录：使用当前工作目录（即 Cargo.toml 所在目录），不要硬编码路径。

若可执行文件不存在，输出以下信息并退出：
```
找不到 rss-reader 可执行文件
请先构建项目：cargo build --release
```

### 2. 拉取最新数据

输出进度提示：
```
正在拉取最新 RSS...
```

执行命令（在项目根目录下）：
```bash
<RSS_READER> fetch
```

解析输出，提取成功数量。若完全失败（0 篇新文章），输出提示但继续执行（数据库中可能有历史文章可用）。

### 3. 获取文章列表（第一轮：标题筛选池）

执行命令，获取最新 500 篇文章的元数据（不含正文内容）：
```bash
<RSS_READER> articles --json 500
```

解析 JSON 输出。若文章列表为空，输出以下信息并退出：
```
数据库中没有文章
建议：检查 RSS 源是否正确配置
运行：<RSS_READER> list
```

### 4. 第一轮 AI 筛选：从 500 标题中选出 Top 100

输出进度提示：
```
正在从 500 篇文章中筛选 Top 100...
```

使用以下提示词分析文章标题：

```
从以下 500 篇 RSS 文章标题中，筛选出最重要的 100 篇。

筛选标准（按优先级）：
1. 重大技术突破、产品发布、安全事件、漏洞披露
2. AI/LLM 领域重要进展
3. 行业影响力大的新闻（大公司动态、融资、政策）
4. 有深度的技术文章（架构设计、性能优化、最佳实践）
5. 开源项目重要更新
6. 值得关注的工具/资源推荐

排除：
- 日常更新、小修小补、changelog 式内容
- 重复/高度相似的报道（同一事件只保留最佳来源）
- 纯广告/营销内容
- 琐碎的个人博客日常

注意分类均衡：确保各领域（安全、AI、技术博客、周刊、资讯、社区、资源等）都有合理代表，不要被单一高产源淹没。

只返回选中文章的 ID 列表，JSON 格式：
{"selected_ids": [123, 456, ...]}

文章数据：
<文章 JSON 数据（仅含 id, title, feed_id, published 字段）>
```

解析 AI 返回的 ID 列表。若筛选后数量不足 100，保留实际数量。

### 5. 获取 Top 100 的完整内容

构造选中文章的 ID 列表字符串（逗号分隔）。

执行命令：
```bash
<RSS_READER> articles --json --with-content --ids=<id列表>
```

解析 JSON 输出。

### 6. 第二轮 AI 分析：生成摘要

输出进度提示：
```
正在生成摘要...
```

使用以下提示词生成最终摘要：

```
为以下 100 篇重要 RSS 文章生成简短摘要。

要求：
- 每篇用一句话概括核心内容（15-30 字）
- 按重要性排序（最重要的排最前）
- 标注所属分类（使用 feed 的 category）

返回 JSON 格式：
{
  "articles": [
    {
      "id": 123,
      "category": "security",
      "title": "原标题",
      "link": "原文链接",
      "summary": "一句话概括",
      "importance": 1
    }
  ]
}

文章数据：
<文章 JSON 数据（含 content）>
```

解析 AI 返回的 JSON。

### 7. 格式化输出

```
今日重要 RSS 摘要 (YYYY-MM-DD)

• <摘要>

  链接: <link>

• <摘要>
  来源: <title>
```

### 8. 输出到 Markdown 文件

在终端展示摘要后，将内容保存到 `rss/YYYY-MM-DD.md` 文件（项目根目录下的 `rss/` 目录）。

创建目录（若不存在）：
```bash
mkdir -p rss
```

检查同名文件是否存在。若存在，追加序号：`YYYY-MM-DD-2.md`、`YYYY-MM-DD-3.md`。

**重要：文件使用标准 Markdown 格式，不要使用终端格式。** 终端输出和文件输出的格式不同：

| 元素 | 终端输出 | Markdown 文件 |
|------|---------|--------------|
| 分隔线 | `━━━━━` | `---` |
| 列表项 | `• 摘要文本` | `- 摘要文本` |
| 分类标题 | `🔒 **安全资讯**` | `## 🔒 安全资讯` |
| 来源/链接 | 两行缩进文本 | `- 摘要 - [来源名](URL)` |

使用 Write 工具写入文件，格式如下：

```markdown
---
date: YYYY-MM-DD
total: N
categories: M
sources: 153+
---

# 今日重要 RSS 摘要 (YYYY-MM-DD)

## 🔒 安全资讯 (N 条)

- 摘要文本 - [来源标题](链接URL)
- 摘要文本 - [来源标题](链接URL)

---

## 🤖 AI 相关 (N 条)

- 摘要文本 - [来源标题](链接URL)

---

（更多分类...）

---

> 共筛选出 **N 条**重要消息（从 500 篇文章中）
> 覆盖 M 个分类，来自 153+ 个 RSS 源

**今日重点关注**：

1. 重点 1
2. 重点 2
```

输出保存确认：
```
✓ 摘要已保存到 rss/YYYY-MM-DD.md
```

## 错误处理

- 所有命令执行失败：显示错误信息并退出
- JSON 解析失败：显示原始输出，提示版本不兼容
- 超时（10 分钟）：自动终止并提示
- 第一轮筛选返回格式异常：尝试从返回文本中提取 ID 数组

## 注意事项

- **工具使用**：使用 Bash tool 执行所有命令
- **工作目录**：所有命令都在项目根目录（Cargo.toml 所在目录）下执行，不要硬编码路径
- **跨平台**：Windows 上使用 `.\target\release\rss-reader.exe`，Linux/macOS 上使用 `./target/release/rss-reader`
- **两轮筛选**：第一轮只用标题快速筛选（减少数据量），第二轮用完整内容生成准确摘要
- **均衡覆盖**：第一轮筛选时注意分类均衡，避免高产源（如新闻聚合类）挤掉低频但高质量的源
- **超时控制**：整个流程应在 10 分钟内完成（源数量多，fetch 耗时较长）
- **数据量控制**：第一轮获取 500 篇标题元数据约 50-100KB，可控；第二轮获取 100 篇含 content 数据量较大但可接受

## Common Mistakes

**错误 1：跳过错误检查**
- 问题：直接执行命令不检查可执行文件是否存在
- 修复：始终先检查可执行文件是否存在（Windows: `.exe` 后缀）

**错误 2：不处理空结果**
- 问题：文章列表为空时继续执行导致错误
- 修复：每个步骤都检查结果，空结果时提前退出并给出提示

**错误 3：单轮处理海量数据**
- 问题：试图一次性获取所有文章的完整内容进行分析
- 修复：必须两轮筛选——第一轮用标题过滤，第二轮才获取内容

**错误 4：分类不均衡**
- 问题：高产源（如新闻聚合）占据大部分名额，低频高质源被忽略
- 修复：第一轮筛选时明确要求分类均衡覆盖

**错误 5：忽略 JSON 解析失败**
- 问题：假设 JSON 总是有效的
- 修复：捕获解析错误，显示原始输出并提示版本不兼容
